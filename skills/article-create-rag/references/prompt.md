# 文章生成提示词规范

## 核心原则

1. **基于证据**：所有观点和案例必须来自提供的素材（外部调研 + 本地文章库）
2. **结构清晰**：遵循经典文章结构（问题引入 → 核心观点 → 分论点 → 案例 → 总结）
3. **观点整合**：将多个来源的观点有机融合，避免简单堆砌
4. **引用标注**：重要观点和金句必须标注来源
5. **语言自然**：避免"AIAI味"，用真人的表达方式

## 文章结构

### 标准结构
````markdown
# [标题]

## 01 问题引入
用具体场景或案例引出问题，引发读者共鸣。

## 02 核心观点
一句话概括文章主旨。

## 03 分论点1
### 观点阐述
[观点内容]

### 案例支撑
[来自素材的案例]
来源：[作者/公众号]

## 04 分论点2
...

## 05 分论点3
...

## 06 总结/行动建议
[总结核心观点，给出可落地的建议]
````

### 变体结构（根据内容调整）
- 列表型：核心观点直接用1/2/3列举
- 问答型：直接用对话形式
- 故事型：用个人经历开场

## 语言风格

### 避免
- "总之"、"综上所述"、"值得一提的是"等AI味连接词
- "我们应该"、"大家需要"等说教式表达
- 过度使用"然而"、"但是"、"此外"等转折词
- 排比句式过度整齐

### 使用
- 口语化的短句
- "我观察到..."、"我有一个想法..."等个人化表达
- 具体数字和案例，而非抽象概念
- 反问句引导思考

## 引用格式

### 正文引用
````markdown
...正如[刘润]所说："[金句内容]"...
````

### 来源标注
````markdown
...
来源：刘润《文章标题》
````

## 素材处理

### 外部调研内容
- 优先提取：新趋势、数据、案例
- 用法：作为论据支撑核心观点

### 本地文章库内容
- 优先提取：经典观点、金句、方法论
- 用法：深化观点、增加说服力

### 整合原则
1. 先确定核心观点（外部调研或本地库）
2. 用另一类素材补充论据
3. 用金句点睛
4. 确保逻辑连贯，无冗余

## 检查清单

生成文章前检查：
- [ ] 每个观点都有素材支撑？
- [ ] 重要金句标注了来源？
- [ ] 结构清晰，有明确的论点？
- [ ] 语言自然，无AI味？
- [ ] 有具体的案例或数据？
- [ ] 结尾有行动建议或总结？
