# 内容缺口分析清单 (Content Gap Analysis Checklist)

本文档定义竞品内容缺口分析的检查项目和标准。

---

## 分析维度

### 1. 主题覆盖度

**检查项目：**
- [ ] 竞品覆盖的主要主题清单
- [ ] 你的内容库相同主题数量
- [ ] 未覆盖的高价值主题识别
- [ ] 覆盖度百分比计算

**覆盖类型：**
- **完全覆盖**：你已有深度内容，竞品只是浅层提及
- **部分覆盖**：你有基础内容，但缺少深度或角度
- **未覆盖**：竞品有而你完全没有的内容

### 2. 关键词使用

**检查项目：**
- [ ] 竞品关键词密度分析
- [ ] 你未使用的高价值关键词
- [ ] 长尾关键词机会识别
- [ ] 语义缺口分析（相同概念的不同表述）

**关键词类型：**
- 核心术语（行业标准词汇）
- 趋势词汇（新兴热词）
- 用户痛点词（用户搜索的词汇）
- 行动词汇（如"如何"、"最佳实践"）

### 3. 内容格式

**检查项目：**
- [ ] 竞品使用的格式类型
- [ ] 你未尝试但有潜力的格式
- [ ] 格式创新机会

**格式类型：**
- 长文深度文章
- 系列文章（1、2、3...）
- 图文结合
- 视频内容
- 互动式内容（代码沙箱、演示）
- 数据可视化（图表、信息图）

### 4. 深度层次

**检查项目：**
- [ ] 入门级内容对比
- [ ] 进阶级内容对比
- [ ] 专家级内容对比
- [ ] 实战案例对比

**深度指标：**
- 概念解释深度
- 实现细节程度
- 最佳实践覆盖
- 常见问题解答

### 5. 更新频率

**检查项目：**
- [ ] 竞品内容更新周期
- [ ] 你的内容更新周期
- [ ] 过时内容识别

**更新标准：**
- 技术跟踪（跟踪新 API、版本）
- 行业动态（政策变化、市场变动）
- 用户反馈（基于评论、提问迭代）

---

## 机会评分标准

每个内容缺口按以下维度评分（1-10 分）：

### 1. 市场需求 (3 分)
- **3 分**：多竞品覆盖，用户搜索需求高
- **2 分**：部分竞品覆盖，中等需求
- **1 分**：少数竞品提及，低需求
- **0 分**：无明显需求

### 2. 竞争优势 (2 分)
- **2 分**：竞品覆盖角度有限，有独特空间
- **1 分**：竞品有强内容，但可提供更好视角
- **0 分**：竞品已充分覆盖，难以差异化

### 3. 内容难度 (2 分)
- **2 分**：需要技术深度或专业知识
- **1 分**：中等难度，可快速产出
- **0 分**：难度低，容易进入（可能竞争激烈）

### 4. 时效性 (2 分)
- **2 分**：信息快速变化，需快速响应
- **1 分**：信息相对稳定，适中周期
- **0 分**：长效内容，低时效性

### 5. SEO 价值 (1 分)
- **1 分**：有 SEO 潜力，但竞争度低
- **0 分**：无 SEO 优势

## 差异化角度建议

基于分析结果，提供以下类型差异化：

### 1. 受众差异化
- 面向不同技能层的内容
- 针对不同行业角色的内容
- 提供更平易理解的解释

### 2. 方法论差异化
- 从工程化角度讲解
- 从成本优化角度分析
- 从安全合规角度探讨
- 从用户体验角度对比

### 3. 案例差异化
- 使用原创案例研究
- 提供企业实际部署经验
- 分享失败案例和教训
- 对比多个方案优劣

### 4. 结构差异化
- 采用交互式内容（代码+演示）
- 使用渐进式教程（从入门到高级）
- 创建专题系列（系统性覆盖）
- 提供完整资源列表（工具、文档、社区）

## 分析流程

1. **数据收集**
   - 爬取竞品内容
   - 提取主题、关键词、格式
   - 记录更新时间

2. **对比分析**
   - 对比主题覆盖度
   - 分析关键词使用差异
   - 评估深度层次

3. **机会识别**
   - 计算机会评分
   - 生成差异化建议
   - 优先级排序

4. **报告生成**
   - 结构化输出 JSON
   - 提供可执行建议
