# 趋势分析框架 (Trend Analysis Frameworks)

本文档定义内容创作调研中趋势分析的方法和标准。

---

## 趋势检测方法

### 1. 弱信号识别

弱信号是趋势早期阶段的表现，具有以下特征：

**识别标准：**
- **低讨论频次**：早期阶段，讨论量低但质量高
- **垂直社区出现**：在小众、专业社区首先出现
- **技术博客预告**：开发者提前分享新工具或方法
- **GitHub 活跃**：新项目突然获得 stars 或 forks
- **会议预告**：行业会议、hackathon 的议题

**信号来源：**
- Reddit (r/LocalLLaMA, r/Machinelearning 等)
- Hacker News
- GitHub Trending
- Twitter/X 技术圈
- Dev.to / Medium 前沿文章
- 行业会议议程

### 2. 增长率计算

**公式：**
```
growth_rate = (current_engagement - baseline_engagement) / baseline_engagement * 100%
```

**指标：**
- 讨论量增长（帖子数、评论数）
- 互动率（点赞/分享比例）
- 搜索量变化
- 提及次数（被其他内容引用）

**时间窗口：**
- `week`: 7 天
- `month`: 30 天
- `quarter`: 90 天

### 3. 时间窗口预测

**预测方法：**
- **线性外推**：基于过去 2-3 个时间点
- **季节性调整**：考虑行业周期性特征
- **事件触发**：会议、产品发布等事件对趋势的推动

**置信度：**
- `high`: 多源一致预测，有历史模式支持
- `medium`: 单一来源预测，模式不明确
- `low`: 缺乏足够历史数据

---

## 信号强度分类

| 强度 | 标准 | 适合内容 |
|------|--------|----------|
| **high** | 增长率 >50%，多源验证，高垂直社区讨论 | 前沿教程、深度解析 |
| **medium** | 增长率 20-50%，单一来源讨论 | 行业评论、应用案例 |
| **low** | 增长率 <20%，讨论分散 | 次要提及、背景介绍 |

---

## 趋势验证清单

在确认趋势前，进行以下验证：

- [ ] 多个来源独立验证
- [ ] 增长趋势持续 2 周以上
- [ ] 有明确的实际应用案例
- [ ] 非短期炒作（检查 4 周以上持续度）
- [ ] 与行业大方向一致
- [ ] 有可量化的数据支撑

## 领域特定模板

### AI 领域

**关键观察点：**
- 模型性能基准
- 新工具/框架发布
- 企业采用案例
- 开发者讨论热点
- 安全和合规议题

**数据源：**
- LLM Leaderboard
- Papers with Code
- Reddit r/LocalLLaMA
- AI industry reports

### 保险领域

**关键观察点：**
- 数字化理赔效率
- 智能核保创新
- 用户隐私与数据主权
- 监管政策变化
- 产品形态创新

**数据源：**
- 行业研报
- 监管部门公告
- 保险科技媒体
- 用户调研报告
