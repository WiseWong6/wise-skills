# Content Brief 模板 (Content Brief Templates)

本文档提供不同内容类型的 Brief 模板和最佳实践。

---

## 通用 Brief 结构

所有 Brief 应包含以下核心部分：

1. **内容概览** - 标题、类型、受众、目标
2. **背景信息** - 趋势、竞品、行业动态
3. **研究支撑** - 带引用的权威资料
4. **结构建议** - 详细大纲和要点
5. **SEO 要素** - 关键词、元数据、内链策略
6. **成功指标** - 可量化的成功标准

## 文章 Brief 模板

### 标题建议模式

基于主题生成吸引力的标题：

```markdown
# [关键词]：[核心主题]的全面解析与实践指南

# [行动动词][主体][价值主张]：[补充说明]

# 如何[解决痛点]：[核心方法]深度指南

# [主题] vs [替代方案]：全面对比分析
```

### 大纲结构

```markdown
## 1. 引言与背景（200-300 字）
- 痛点或问题引入
- 说明为什么这个话题重要
- 提及趋势背景

## 2. 核心概念解析（500-800 字）
- 清晰定义核心概念
- 使用类比或案例帮助理解
- 引用权威资料

## 3. 实际应用/案例（800-1200 字）
- 1-2 个具体应用场景
- 包含真实数据或结果
- 引用差异化案例

## 4. 对比分析（400-600 字）
- 与竞品或传统方案对比
- 突出优势
- 强调独特视角

## 5. 实施指南/最佳实践（600-900 字）
- 可操作的建议或步骤
- 注意事项和常见陷阱
- 工具或资源推荐

## 6. 总结与展望（200-300 字）
- 总结关键要点
- 提出未来发展方向
- 引导读者思考或行动
```

## 教程 Brief 模板

### 标题建议

```markdown
# [主题] 教程：从入门到精通

# [主题] 完全指南（20XX 版）

# 如何用 [主题] 构建 [应用类型]
```

### 大纲结构

```markdown
## 1. 前置知识准备
- 必备概念和技能
- 环境要求说明
- 学习路径建议

## 2. 环境搭建
- 详细的安装/配置步骤
- 验证安装成功
- 常见问题解决

## 3. 核心概念讲解
- [主题] 的基础原理和机制
- 架构图解（如适用）
- 关键 API 或配置说明

## 4. 实践步骤
- 手把手代码示例
- 每个步骤的预期输出
- 调试和验证方法

## 5. 常见问题与解决
- FAQ 格式的故障排除
- 错误消息解释
- 最佳规避方案

## 6. 进阶技巧与优化
- 面向有经验的内容
- 性能优化建议
- 集成和扩展方法

## 7. 资源与延伸阅读
- 官方文档链接
- 推荐相关库和工具
- 社区资源
```

## 案例研究 Brief 模板

### 标题建议

```markdown
# [公司/项目] 如何用 [主题] 实现 [目标]

# [主题] 应用案例：从挑战到成果

# [价值主张]：[量化数据与业务影响]
```

### 大纲结构

```markdown
## 1. 背景介绍
- 企业/项目背景
- 面临的具体挑战
- 预期成果和目标

## 2. 挑战与目标
- 具体难题描述
- 量化指标（时间、成本、效率）
- 成功定义标准

## 3. 解决方案实施
- [主题] 的应用过程
- 技术选型和架构决策
- 实施步骤和时间线

## 4. 结果与影响
- 量化数据（提升比例、节省成本、时间缩短）
- 质量改进（错误率、用户满意度）
- 业务价值（收入影响、市场份额）

## 5. 关键经验与教训
- 实施过程中的挑战
- 解决方案回顾
- 可复用的洞察

## 6. 最佳实践建议
- 后续优化方向
- 类似场景应用建议
- 资源和工具推荐
```

## SEO 最佳实践

### 标元数据

```markdown
**标题**: 50-60 字符，包含主关键词
**描述**: 前 150 字符，包含关键词和行动号召
**URL**: 英文、短、含主关键词（如 `ai-agent-best-practices`）
```

### 内容 SEO

```markdown
- 主关键词出现在 H1
- 次关键词出现在 H2/H3
- 首段前 200 字出现关键词
- 内链到权威来源
- 图片包含 alt 文本
- 合理的关键词密度（2-3%）
```

### 内链策略

- **枢纽内容**: 链接到核心主题页
- **系列链接**: 链接到前后关联内容
- **权威外链**: 链接到官方文档、论文等
- **相关资源**: 链接到工具、社区资源

## 成功指标模板

### 阅读指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 目标阅读时间 | 3-5 分钟 | Google Analytics 平均停留时间 |
| 平均阅读完成率 | >70% | 滚动至底部比例 |
| 重复访问率 | >20% | 7 天内回访用户比例 |
| 页面价值得分 | >80 | 内容质量评分 |

### 互动指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 社交分享数 | 基于趋势热度 | 社媒分享次数 |
| 互动率 | >5% | 点赞、评论、转发比例 |
| 评论质量 | 高质量讨论 | 评论深度和有用性 |
| 引用/提及 | 被其他内容引用 | 外链和提及次数 |

### SEO 指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 主关键词排名 | Top 10 | Google Search Console |
| 搜索展现量 | >1000 次月 | GSC 数据 |
| 点击率 (CTR) | >3% | GSC 数据 |
| 精选片段 | 获得精选 | Google 搜索结果 |

## Brief 质量检查清单

在发布 Brief 前，确认以下项目：

**内容完整性**
- [ ] 标题明确且吸引人
- [ ] 目标受众清晰定义
- [ ] 核心目标明确
- [ ] 趋势背景已整合
- [ ] 研究资料已引用

**结构质量**
- [ ] 大纲逻辑清晰
- [ ] 每个部分有具体字数目标
- [ ] 包含实际案例或示例
- [ ] 有明确的行动号召

**SEO 优化**
- [ ] 主关键词确定
- [ ] 次关键词列表完整
- [ ] Meta 描述已撰写
- [ ] 内链策略已定义
- [ ] 成功指标可量化

**差异化价值**
- [ ] 与竞品对比明确
- [ ] 独特角度已识别
- [ ] 缺口分析已整合
- [ ] 价值主张清晰

## 常见错误

**避免：**
- 过于笼统的标题（如"AI 入门"）
- 缺乏具体目标受众
- 无研究支撑的观点
- 无限长的段落
- 缺少视觉元素

**确保：**
- 每个主张有数据支持
- 引用来源可访问
- 例子可运行/验证
- 时间敏感信息已标注
- 多个角度被考虑

## 资讯合集模板 (News Digest Template)

### 标题建议

```markdown
# {主题} 资讯速递 - {YYYY-MM-DD}
# AI Agent 领域最新资讯（第 {N} 期）
# 本周 {主题} 动态汇总
```

### 大纲结构

```markdown
## 📊 资讯概览
- **主题**：{主题}
- **时间范围**：近 {N} 天
- **资讯数量**：{总数} 条
- **平均评分**：{平均分}/10

## 🏆 热门资讯（Top 10）
### {排名}. {标题}
**评分**：{score}/10
- 创新性：{innovation}/10
- 实用性：{practicality}/10
- 影响力：{impact}/10
**摘要**：{中文摘要}
**来源**：[{来源名称}]({URL})
**日期**：{YYYY-MM-DD}

## 📂 分类展示
### 🚀 新技术与产品
### 📚 教程与指南
### 📊 研究与报告
### 💡 行业动态

## 📈 趋势洞察
基于本次资讯搜集，观察到以下趋势...

## 🔗 完整列表
| 评分 | 标题 | 来源 | 日期 |
```

### 评分标准

| 维度 | 权重 | 评分规则 |
|------|------|----------|
| 创新性 | 30% | 基础 5 分，关键词（new/breakthrough/最新/突破）各 +1 分 |
| 实用性 | 30% | 基础 5 分，关键词（tutorial/guide/教程/指南）各 +1 分 |
| 影响力 | 40% | 官方来源 10 分，高置信度 9 分，中置信度 7 分 |

### 分类规则

- **🚀 新技术与产品**：包含 "new", "release", "launch", "发布", "新品" 等关键词
- **📚 教程与指南**：包含 "tutorial", "guide", "how-to", "教程", "指南" 等关键词
- **📊 研究与报告**：包含 "research", "paper", "study", "研究", "报告", "论文" 等关键词
- **💡 行业动态**：其他类型归为此类
